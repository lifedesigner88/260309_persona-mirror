- Replaced the legacy recommendation response with a conversation-priority response shaped around `safe_fit`, `complementary_fit`, and `wildcard_fit`.
- Scored candidates with interpretable factors: problem resonance, role complementarity, work-style compatibility, value alignment, and conversation potential.
- Restored the frontend recommendation area with three explanation-first cards, a first-question suggestion, uncertainty notes, and a low-signal candidate list.
- Relaxed slot selection so the recommendation engine now fills the three conversation-priority slots whenever the approved candidate pool has enough usable signal, while still marking weaker picks with stronger uncertainty notes.
- Turned the `left for later` candidate list into clickable detail cards so lower-ranked candidates can still be inspected for MBTI, SDGs, problem framing, and links.
- Removed the separate `answer one more AI question` panel from the root saved-results area so the page focuses on the saved profile and recommendation cards, while follow-up answering stays inside the profile detail flow.
- Replaced the separate saved-profile summary modal with a tabbed post-save workspace: `Candidates` opens first after saving, and `Team-fit profile` now exposes the saved intake fields plus editable interview transcript directly on the page.
- Expanded the explorer save contract so an already-saved profile can overwrite its full interview history, letting the frontend re-save transcript edits instead of staying read-only after the initial 3-turn flow.
- Removed the extra guest/member status notice above the main team-fit card so the root page now enters the tabbed workspace more directly without an extra preview message block.
- Removed the extra `Interview profile` badge above the main card so the page now starts more directly with the tabbed workspace itself.
- Flattened the tabbed `Team-fit profile` workspace so it is now the top-level card itself instead of a second rounded card nested inside another `ShellCard`.

- Added the same `0~100` fit-score + memo editor to signed-up-user detail cards, so even non-recommended people can be scored after a real conversation and then excluded from future recommendation rounds.
- Unified the signed-up-user detail modal with the recommendation modal by showing the same 5 recommendation criteria cards there too, with graceful fallback copy when a person has no saved team-fit profile.
- Removed the extra `team-fit state` and `saved fit record` summary cards from the signed-up-user detail modal because they duplicated the new 5-criteria grid and fit editor.
- Added a softer hero-like gradient to the main interview-profile workspace card so the top hero and the editable team-fit area feel more visually connected.
- Removed the extra `Interview profile` heading and repeated edit-description copy from the main workspace so the card now starts directly with the tab controls and content.
- Simplified the saved interview transcript cards into read-only `Q.` / `A.` blocks, removing the old inline textarea editing UI while keeping the existing save contract unchanged.
- Compressed each saved interview turn further into a single simple card that renders only `Q : ...` and `A : ...` lines.
- Matched the recommendation/detail interview-history dialog to that same simple transcript format, replacing the older phase badges and nested question/answer cards with a single `Q : ...` / `A : ...` card per turn.
- Fixed the root-page recommendation refresh path after profile edits so saving within the same timestamp window still triggers a fresh recommendation fetch instead of leaving the candidate tab empty.
- Relaxed the fallback candidate selector so low-text-overlap but still high-signal profiles can refill the 3 recommendation slots instead of dropping all cards when `problem_resonance` is unusually low.
- Rewrote the recommendation-algorithm modal cards into a more developer-readable explanation of the current implementation: signal extraction, embedding retrieval, weighted scoring, and the final safe/complementary/wildcard slot selection.
- Expanded the demo seed from 6 to 12 approved explorer profiles so the recommendation engine has a wider candidate pool and is less likely to return empty or overly repetitive conversation-priority cards during demos.
- Fixed the explorer schema startup patch so Postgres uses a compatible timestamp type for `extracted_at`, and switched startup column checks away from `vector` reflection warnings.
- Added local draft persistence for the team-fit intake so guest-entered Step 1 / Step 2 values survive login and restore until a real profile is saved.
- Extended the demo seed so the existing 6 demo users now also receive explorer profiles, 3-turn interview transcripts, and fresh recommendation artifacts for the new recommendation engine.
- Turned the saved-profile area into a compact summary card and moved the full profile plus transcript into a detail modal opened on click.
- Updated the recommendation algorithm modal and both READMEs so the product now explicitly frames recommendations as “the first people worth talking to,” not “the best teammates.”

### Why It Mattered

- The recommendation layer now matches the new interview-first data model instead of staying disconnected behind a placeholder.
- Users can move from profile writing straight into real conversation decisions without the product overclaiming certainty.
- The system now prefers interpretability and conversation value over opaque top-3 similarity ranking.
- The recommendation surface now stays closer to the intended product promise: show three people to talk to first, then let users inspect the deferred candidates without losing context.
- The recommendations tab no longer ends in explanatory copy alone; it now doubles as a broader discovery surface for browsing the whole signed-up pool after the first three candidates.
- The post-save experience now behaves like an editable workspace instead of a one-way result screen, which makes profile cleanup and transcript corrections much easier during demos.
